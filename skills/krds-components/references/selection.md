# Selection 컴포넌트

## 라디오 버튼 (Radio Button)

### 용도

상호 배타적인 선택지 중 하나를 선택하는 컴포넌트. 2~7개 정도의 선택지가 있을 때 적합하다.

### HTML 스니펫

```
html/code/radio_button.html
html/code/radio_chip.html
html/code/radio_size.html
```

### 구조

- 라디오 그룹 (`<fieldset>`)
  - 그룹 제목 (`<legend>`)
  - 라디오 항목
    - 라디오 입력 (`<input type="radio">`)
    - 레이블 (`<label>`)

### 유형

| 유형 | 파일 | 용도 |
|------|------|------|
| 기본 라디오 | `radio_button.html` | 일반적인 라디오 선택 |
| 칩 라디오 | `radio_chip.html` | 칩 형태의 라디오 선택 |

### 사용성 가이드라인

1. 라디오 버튼은 복수 선택이 불가능하다. 복수 선택이 필요하면 체크박스를 사용한다.
2. 선택지가 너무 많을 경우(7개 초과) 라디오 버튼 대신 셀렉트(드롭다운)를 사용한다.
3. 선택지가 많을 때 가나다순(또는 알파벳순)으로 정렬한다.
4. 라디오 버튼은 수직으로 배치하는 것을 기본으로 한다. 가독성이 높다.
5. 레이블은 라디오 버튼의 우측에 배치한다.
6. 수평 배치 시 각 항목 사이에 충분한 간격을 확보한다 (최소 16px).
7. 기본 선택값을 미리 지정하지 않는다. 사용자가 의도적으로 선택하도록 한다 (단, 대부분의 사용자가 동일한 값을 선택하는 경우 예외).
8. 선택을 되돌릴 수 있는 방법을 제공한다 (초기화 버튼 등).
9. **탐색 상호작용**: 포커스/호버 시 시각적 피드백을 제공한다.
10. **실행 상호작용**: 클릭 또는 Space 키로 선택한다.

### 접근성 가이드라인

- `<fieldset>`과 `<legend>`로 라디오 그룹을 감싼다.
- 각 라디오 입력에 `<label>`을 연결한다.
- 같은 그룹의 라디오 버튼은 동일한 `name` 속성을 사용한다.
- Arrow 키로 그룹 내 항목 간 이동을 지원한다.
- KWCAG 2.2 / WCAG 2.1 준수: 1.3.1 Info and Relationships, 4.1.2 Name, Role, Value

### 상호작용

- **Tab**: 라디오 그룹에 포커스 (선택된 항목 또는 첫 번째 항목)
- **Arrow Up/Down**: 그룹 내 항목 간 이동 및 선택
- **Space**: 현재 포커스된 항목 선택
- **클릭**: 해당 항목 선택

### 플랫폼 고려사항

- 모바일에서 터치 영역을 충분히 확보한다 (레이블 영역 포함 44px 이상).
- 칩 형태(`radio_chip.html`)는 가로 스크롤이 가능하도록 한다.

## 셀렉트 (Select)

### 용도

드롭다운 목록에서 하나의 옵션을 선택하는 컴포넌트. 선택지가 많을 때(7개 초과) 라디오 버튼 대신 사용한다.

### HTML 스니펫

```
html/code/select.html
html/code/select_size.html
html/code/select_sorting.html
html/code/select_state.html
```

### 구조

- 셀렉트 그룹
  - 레이블 (`<label>`)
  - 셀렉트 요소 (`<select>` 또는 커스텀 드롭다운)
    - 옵션 항목 (`<option>`)
  - 오류 메시지 (선택)

### 사용성 가이드라인

1. 셀렉트는 복수 선택을 지원하지 않는다. 복수 선택이 필요하면 체크박스를 사용한다.
2. 선택지가 적을 경우(7개 이하) 셀렉트 대신 라디오 버튼을 사용한다.
3. 기본 선택값 또는 안내 텍스트를 표시한다 (예: "선택하세요").
4. 모든 셀렉트에 시각적 레이블을 제공한다.
5. 값 변경 시 자동으로 폼을 제출하거나 페이지를 이동하지 않는다. 사용자가 별도로 실행 버튼을 클릭해야 한다.
6. **확장/축소 상호작용**: 셀렉트 클릭 시 드롭다운 열림/닫힘.
7. **탐색 상호작용**: Arrow 키로 옵션 간 이동.
8. **선택 상호작용**: Enter 또는 클릭으로 옵션 선택.

### 접근성 가이드라인

- `<label for="id">`로 레이블과 셀렉트를 연결한다.
- 커스텀 셀렉트 사용 시 `role="listbox"`, `role="option"`, `aria-expanded` 등을 적용한다.
- 필수 선택에 `required` 속성을 적용한다.
- 오류 메시지를 `aria-describedby`로 연결한다.
- KWCAG 2.2 / WCAG 2.1 준수: 4.1.2 Name, Role, Value, 3.2.2 On Input

### 상호작용

- **클릭/Enter/Space**: 드롭다운 열기
- **Arrow Up/Down**: 옵션 간 이동
- **Enter**: 옵션 선택 및 드롭다운 닫기
- **Esc**: 드롭다운 닫기 (선택 취소)
- **Tab**: 다음 대화형 요소로 포커스 이동

### 플랫폼 고려사항

- 모바일에서 네이티브 `<select>`를 사용하면 OS 기본 피커가 표시된다.
- 커스텀 셀렉트 사용 시 모바일 터치 영역을 충분히 확보한다.

## 체크박스 (Checkbox)

### 용도

여러 선택지 중 하나 이상을 선택하거나 개별 옵션의 동의/비동의를 표시하는 컴포넌트.

### HTML 스니펫

```
html/code/checkbox.html
html/code/checkbox_chip.html
html/code/checkbox_size.html
```

### 구조

- 체크박스 그룹 (`<fieldset>`, 그룹인 경우)
  - 그룹 제목 (`<legend>`)
  - 체크박스 항목
    - 체크박스 입력 (`<input type="checkbox">`)
    - 레이블 (`<label>`)
  - 전체 선택 체크박스 (선택)

### 유형

| 유형 | 파일 | 용도 |
|------|------|------|
| 기본 체크박스 | `checkbox.html` | 일반적인 체크 선택 |
| 칩 체크박스 | `checkbox_chip.html` | 칩 형태의 체크 선택 |

### 사용성 가이드라인

1. 단일 선택만 필요한 경우 체크박스 대신 라디오 버튼을 사용한다.
2. 선택지가 많을 때 가나다순(또는 알파벳순)으로 정렬한다.
3. 체크박스는 수직으로 배치하는 것을 기본으로 한다.
4. 레이블은 체크박스의 우측에 배치한다.
5. "전체 선택" 체크박스 사용 시, 일부만 선택된 상태에서는 중간 상태(indeterminate)를 표시한다.
6. **탐색 상호작용**: 포커스/호버 시 시각적 피드백을 제공한다.
7. **선택/해제 상호작용**: 클릭 또는 Space 키로 선택/해제한다.

### 접근성 가이드라인

- 그룹 체크박스는 `<fieldset>`과 `<legend>`로 감싼다.
- 각 체크박스에 `<label>`을 연결한다.
- 중간 상태(indeterminate)는 JavaScript로 `indeterminate` 속성을 설정하고, `aria-checked="mixed"`를 적용한다.
- KWCAG 2.2 / WCAG 2.1 준수: 1.3.1 Info and Relationships, 4.1.2 Name, Role, Value

### 상호작용

- **Tab**: 체크박스 항목 간 포커스 이동
- **Space**: 선택/해제 토글
- **클릭**: 선택/해제 토글 (레이블 클릭 포함)

### 플랫폼 고려사항

- 모바일에서 터치 영역을 충분히 확보한다 (레이블 영역 포함 44px 이상).
- 칩 형태(`checkbox_chip.html`)는 가로 스크롤이 가능하도록 한다.

## 태그 (Tag)

### 용도

콘텐츠를 분류하거나 필터링하기 위한 키워드 레이블. 대화형(선택/삭제 가능)과 비대화형(표시 전용)으로 구분한다.

### HTML 스니펫

```
html/code/tag.html
html/code/tag_link.html
```

### 구조

- 태그 요소 (`krds-tag`)
  - 레이블 텍스트
  - 삭제 버튼 (대화형, 선택)
  - 링크 (링크 태그, 선택)

### 유형

| 유형 | 파일 | 용도 |
|------|------|------|
| 기본 태그 | `tag.html` | 분류/필터 표시 |
| 링크 태그 | `tag_link.html` | 클릭 시 해당 카테고리로 이동 |

### 사용성 가이드라인

1. 태그 레이블은 간결하게 작성한다. 2~3단어 이내를 권장한다.
2. 대화형 태그(선택/삭제 가능)와 비대화형 태그(표시 전용)를 한 영역에서 혼용하지 않는다.
3. **탐색 상호작용**: 포커스/호버 시 시각적 피드백을 제공한다.
4. **실행 상호작용**: 링크 태그 클릭 시 해당 카테고리/검색 결과로 이동한다.
5. **삭제 상호작용**: 삭제 버튼 클릭 시 해당 태그를 제거한다.

### 접근성 가이드라인

- 링크 태그는 `<a>` 태그를 사용한다.
- 삭제 가능한 태그의 삭제 버튼에 `aria-label="태그명 삭제"`를 제공한다.
- 태그 목록 변경 시 `aria-live="polite"` 영역에서 스크린 리더에 알린다.
- KWCAG 2.2 / WCAG 2.1 준수: 4.1.2 Name, Role, Value

### 상호작용

- **Tab**: 태그 항목 간 포커스 이동
- **Enter**: 링크 태그 시 이동, 삭제 버튼 시 삭제
- **Backspace/Delete**: 포커스된 태그 삭제 (선택적)
- **클릭**: 이동 또는 삭제

### 플랫폼 고려사항

- 모바일에서 태그가 여러 줄로 줄바꿈될 수 있다.
- 터치 영역을 충분히 확보한다 (삭제 버튼 포함).
