# Navigation 컴포넌트

## 건너뛰기 링크 (Skip Link)

### 용도

키보드 사용자가 반복되는 내비게이션 영역을 건너뛰고 본문 콘텐츠로 바로 이동할 수 있도록 제공하는 링크.

### HTML 스니펫

```
html/code/skip_link.html
```

### 구조

- 건너뛰기 링크 컨테이너 (`krds-skip-link`)
  - 링크 항목(들) — 최대 3개

### 사용성 가이드라인

1. 반복 영역(헤더, 메뉴)이 없는 화면에서는 건너뛰기 링크를 생략할 수 있다.
2. 반복 영역이 있는 모든 화면에서 건너뛰기 링크를 제공한다.
3. 건너뛰기 링크는 3개 이내로 제한한다. 너무 많으면 오히려 탐색 효율이 떨어진다.
4. 첫 번째 항목은 반드시 "본문 바로가기"로 설정한다.
5. 각 링크의 목적지는 적절한 콘텐츠 영역의 `id`와 연결한다.
6. 포커스 시 화면에 표시되고, 포커스를 잃으면 다시 숨겨진다.

### 접근성 가이드라인

- 건너뛰기 링크는 페이지에서 가장 먼저 포커스를 받는 요소여야 한다 (공식 배너보다도 먼저).
- 시각적으로 숨기되 스크린 리더와 키보드에는 접근 가능하게 구현한다 (CSS `position: absolute` 기법).
- 포커스 시 시각적으로 표시한다 (`:focus` 스타일).
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.1 Bypass Blocks

### 상호작용

- **Tab**: 건너뛰기 링크에 포커스 (화면에 표시됨)
- **Enter**: 해당 목적지로 포커스 이동
- **Tab (포커스 해제 시)**: 링크 숨김

### 플랫폼 고려사항

- 모바일에서도 키보드/보조 기기 사용자를 위해 동일하게 제공한다.

## 메인 메뉴 (Main Menu)

### 용도

사이트의 주요 섹션을 탐색하는 최상위 내비게이션. PC와 모바일에서 다른 형태로 표시된다.

### HTML 스니펫

```
html/code/main_menu_pc.html
html/code/main_menu_mobile.html
```

### 구조

- 내비게이션 컨테이너 (`krds-main-menu`)
  - 1수준 메뉴 항목
    - 2수준 서브메뉴
      - 3수준 서브메뉴 (최대)

### 사용성 가이드라인

1. 메뉴 항목은 좌측 정렬한다.
2. 메뉴 항목이 뷰포트 너비를 초과하지 않도록 한다. 항목이 많을 경우 그룹화한다.
3. 메뉴 항목이 많아 넘칠 경우 가로 스크롤을 허용한다.
4. 현재 활성 메뉴(Active)와 선택된 메뉴(Selected)를 시각적으로 구분한다.
5. 서브메뉴 영역에는 해당 섹션의 제목을 표시한다.
6. 메뉴 계층은 3수준 이하로 제한한다. 깊은 계층은 사용자 혼란을 유발한다.
7. 메뉴 항목 이름은 사용자가 쉽게 이해할 수 있는 용어를 사용한다.
8. 마우스 오버(hover)만으로 서브메뉴를 확장하지 않는다. 클릭으로 확장한다.
9. 1수준 메뉴 항목은 항상 전체를 표시한다. 숨기거나 "더보기"로 축소하지 않는다.
10. 서브메뉴의 확장/축소 상태를 명확하게 표시한다 (화살표 아이콘, aria-expanded).
11. 키보드만으로 모든 메뉴 항목을 탐색할 수 있어야 한다.

### 접근성 가이드라인

- `<nav aria-label="메인 내비게이션">`으로 감싼다.
- 서브메뉴 토글 버튼에 `aria-expanded` 속성을 적용한다.
- 현재 페이지에 해당하는 메뉴 항목에 `aria-current="page"`를 적용한다.
- Arrow 키로 메뉴 항목 간 이동, Enter로 선택, Esc로 서브메뉴 닫기를 지원한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.1 Bypass Blocks, 2.4.5 Multiple Ways, 3.2.3 Consistent Navigation

### 상호작용

- **1수준 메뉴 클릭**: 서브메뉴 확장 또는 해당 페이지 이동
- **2/3수준 메뉴 클릭**: 해당 페이지로 이동
- **Arrow Left/Right**: 1수준 메뉴 항목 간 이동
- **Arrow Down/Up**: 서브메뉴 항목 간 이동
- **Enter**: 메뉴 항목 선택/이동
- **Esc**: 서브메뉴 닫기, 포커스를 상위 메뉴로 이동

### 플랫폼 고려사항

- 모바일에서는 햄버거 버튼을 통해 전체 화면 메뉴로 전환한다 (`main_menu_mobile.html`).
- 모바일 메뉴는 아코디언 형태로 서브메뉴를 펼친다.
- 모바일 메뉴 열림 시 본문 스크롤을 잠근다 (`overflow: hidden`).
- 모바일 메뉴 닫기 시 포커스를 햄버거 버튼으로 복귀시킨다.

## 브레드크럼 (Breadcrumb)

### 용도

사용자의 현재 위치를 계층적으로 표시하는 내비게이션 보조 요소. 상위 페이지로의 이동 경로를 제공한다.

### HTML 스니펫

```
html/code/breadcrumb.html
```

### 구조

- 브레드크럼 컨테이너 (`krds-breadcrumb`)
  - 항목 리스트 (`<ol>`)
    - 홈 링크
    - 중간 계층 링크(들)
    - 현재 페이지 (마지막 항목, 링크 아님)
  - 구분자 아이콘

### 사용성 가이드라인

1. 1수준(메인 페이지) 화면에서는 브레드크럼을 사용하지 않는다.
2. 메인 페이지 및 랜딩 페이지에서는 브레드크럼을 사용하지 않는다.
3. 브레드크럼을 진행 상황(step indicator)으로 사용하지 않는다. 브레드크럼은 계층 구조 표시 전용이다.
4. 각 항목 사이에 시각적 구분자(예: `>`, `/`, `ico_angle.svg`)를 표시한다.
5. 첫 번째 항목은 반드시 메인 페이지(홈)로 설정한다.
6. 마지막 항목은 현재 페이지의 바로 상위 화면을 나타낸다. 현재 페이지 자체는 링크가 아닌 텍스트로 표시한다.
7. 브레드크럼과 본문 제목(H1) 사이에 다른 UI 요소를 배치하지 않는다.
8. 전체 계층 경로를 생략 없이 표시한다.
9. 현재 위치에 대한 시각적 단서(색상, 굵기 등)를 제공한다.
10. 브레드크럼 항목은 4개 이하로 유지한다. 계층이 깊을 경우 중간 계층을 생략할 수 있다.
11. 브레드크럼은 단일 행에 표시한다. 줄바꿈하지 않는다.
12. 좌측 정렬하여 배치한다.
13. 각 항목 사이에 충분한 간격을 확보하여 터치/클릭이 용이하도록 한다.
14. 브레드크럼을 메인 내비게이션(메인 메뉴)의 대체 수단으로 사용하지 않는다.
15. 모바일에서는 첫 번째(홈)와 마지막(현재) 항목만 표시할 수 있다.
16. **탐색 상호작용**: 브레드크럼 항목에 포커스/호버하면 시각적 피드백을 제공한다.
17. **이동 상호작용**: 항목 클릭 시 해당 계층 페이지로 이동한다.

### 접근성 가이드라인

- `<nav aria-label="브레드크럼">`으로 감싼다.
- 항목 목록에 `<ol>` 태그를 사용한다 (순서가 의미 있음).
- 현재 페이지 항목에 `aria-current="page"`를 적용한다.
- 구분자 아이콘에 `aria-hidden="true"`를 적용한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.8 Location

### 상호작용

- **항목 클릭/Enter**: 해당 계층 페이지로 이동
- **Tab**: 브레드크럼 항목 간 포커스 이동
- **호버**: 밑줄 또는 색상 변화로 링크임을 표시

### 플랫폼 고려사항

- 모바일에서 항목이 많을 경우, 첫 번째(홈)와 마지막(현재) 항목만 표시하고 중간을 생략한다.
- 모바일에서도 가로 스크롤 없이 한 줄에 표시되도록 한다.

## 사이드 메뉴 (Side Navigation)

### 용도

섹션 내부의 하위 페이지 탐색을 위한 좌측 내비게이션. 메인 메뉴의 하위 계층을 표시한다.

### HTML 스니펫

```
html/code/side_navigation.html
```

### 구조

- 사이드 메뉴 컨테이너 (`krds-side-navigation`)
  - 메뉴 제목 (현재 섹션명)
  - 1수준 메뉴 항목
    - 2수준 하위 항목 (최대)

### 사용성 가이드라인

1. 현재 활성 페이지를 시각적으로 강조한다 (배경색, 굵은 글씨, 좌측 인디케이터 등).
2. 하위 항목은 2수준까지만 허용한다.
3. 하위 항목이 있는 메뉴는 확장 가능하도록 표시한다 (화살표 아이콘).
4. 계층 간 시각적 구분을 제공한다 (들여쓰기, 구분선 등).
5. 모바일에서는 사이드 메뉴를 숨기고, 필요 시 메인 메뉴에서 접근한다.
6. **탐색 상호작용**: 메뉴 항목에 포커스/호버 시 시각적 피드백을 제공한다.
7. **이동 상호작용**: 항목 클릭 시 해당 페이지로 이동한다.
8. **확장/축소 상호작용**: 하위 메뉴가 있는 항목 클릭 시 하위 목록을 펼치거나 접는다.

### 접근성 가이드라인

- `<nav aria-label="사이드 내비게이션">`으로 감싼다.
- 현재 페이지 항목에 `aria-current="page"`를 적용한다.
- 확장 가능한 항목에 `aria-expanded` 속성을 적용한다.
- 키보드로 모든 항목을 탐색할 수 있어야 한다 (Tab, Arrow, Enter).
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.5 Multiple Ways, 3.2.3 Consistent Navigation

### 상호작용

- **항목 클릭/Enter**: 해당 페이지로 이동 또는 하위 메뉴 확장
- **Tab**: 메뉴 항목 간 포커스 이동
- **Arrow Down/Up**: 같은 수준의 항목 간 이동
- **Arrow Right**: 하위 메뉴 확장
- **Arrow Left**: 하위 메뉴 축소

### 플랫폼 고려사항

- 모바일(small, medium 브레이크포인트)에서는 사이드 메뉴를 숨긴다.
- 모바일 사용자는 메인 메뉴(햄버거)나 브레드크럼을 통해 하위 페이지를 탐색한다.

## 콘텐츠 내 탐색 (In-Page Navigation)

### 용도

긴 페이지 내에서 섹션 간 빠른 이동을 제공하는 목차형 내비게이션. 페이지 우측에 고정 배치된다.

### HTML 스니펫

```
html/code/in_page_navigation.html
```

### 구조

- 탐색 컨테이너 (`krds-in-page-navigation`)
  - 제목 ("목차" 또는 페이지 제목)
  - 섹션 링크 목록
  - 현재 위치 인디케이터

### 사용성 가이드라인

1. 페이지 내 계층적 구분이 없는 단순 페이지에서는 사용하지 않는다.
2. 제목과 현재 섹션 인디케이터를 함께 표시한다.
3. 적절한 너비를 유지한다. 본문 영역을 과도하게 차지하지 않도록 한다.
4. 페이지 우측에 고정 배치한다 (PC 기준).
5. 섹션 링크 텍스트는 실제 섹션 제목(Heading)과 일치시킨다.
6. 목록 항목은 3수준 이하로 제한한다.
7. 모바일에서는 제목과 본문 사이에 가로 형태로 배치한다.
8. **목록 탐색**: 링크에 포커스/호버 시 시각적 피드백을 제공한다.
9. **섹션 이동**: 링크 클릭 시 해당 섹션으로 스크롤 이동한다.

### 접근성 가이드라인

- `<nav aria-label="콘텐츠 내 탐색">`으로 감싼다.
- 현재 보이는 섹션에 해당하는 링크를 시각적으로 강조한다.
- 스크롤 연동 시 `aria-current` 또는 클래스로 현재 섹션을 표시한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.5 Multiple Ways

### 상호작용

- **링크 클릭/Enter**: 해당 섹션으로 스무스 스크롤 이동
- **Tab**: 링크 항목 간 포커스 이동
- **스크롤**: 현재 보이는 섹션에 해당하는 링크가 자동으로 강조됨 (JS 연동)

### 플랫폼 고려사항

- 모바일에서는 우측 고정 대신, 페이지 제목과 본문 사이에 가로 스크롤 형태로 배치한다.
- JS로 스크롤 위치를 감지하여 현재 섹션을 실시간 업데이트한다.

## 페이지네이션 (Pagination)

### 용도

여러 페이지에 걸친 콘텐츠를 탐색하는 컴포넌트. 목록, 검색 결과 등에서 사용한다.

### HTML 스니펫

```
html/code/pagination.html
```

### 구조

- 페이지네이션 컨테이너 (`krds-pagination`)
  - 첫 페이지 버튼
  - 이전 페이지 버튼
  - 페이지 번호 링크 목록
  - 다음 페이지 버튼
  - 마지막 페이지 버튼
  - 전체 건수 표시 (선택)

### 사용성 가이드라인

1. 무한 스크롤과 페이지네이션을 동시에 사용하지 않는다. 하나의 방식만 선택한다.
2. 첫 페이지, 마지막 페이지, 이전 페이지, 다음 페이지 버튼을 모두 제공한다.
3. 전체 항목 수 또는 전체 페이지 수를 표시한다.
4. 현재 페이지 번호를 시각적으로 강조한다 (배경색, 굵은 글씨 등).
5. 목록 하단 중앙에 배치한다.
6. 한 페이지에 페이지네이션은 하나만 사용한다.
7. 페이지 번호는 10개 이내로 표시한다. 초과 시 생략(...)을 사용한다.
8. 모바일에서는 페이지네이션을 수직 방향으로 배치할 수 있다.
9. 모바일에서 표시하는 페이지 번호는 7개 이내로 줄인다.
10. 전체 건수 표시는 페이지 이동 시에도 일정하게 유지한다.
11. **숫자 링크 상호작용**: 번호 클릭 시 해당 페이지로 이동한다.
12. **직접 이동 상호작용**: 페이지 번호를 직접 입력하여 이동할 수 있다 (선택적).
13. **목록 확장 상호작용**: 페이지당 표시 건수를 변경할 수 있다 (선택적).

### 접근성 가이드라인

- `<nav aria-label="페이지네이션">`으로 감싼다.
- 현재 페이지에 `aria-current="page"`를 적용한다.
- 비활성(disabled) 버튼에 `aria-disabled="true"`를 적용한다.
- 이전/다음 버튼에 `aria-label="이전 페이지"`, `aria-label="다음 페이지"`를 제공한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.4 Link Purpose, 4.1.2 Name, Role, Value

### 상호작용

- **페이지 번호 클릭/Enter**: 해당 페이지로 이동
- **이전/다음 버튼 클릭**: 이전/다음 페이지로 이동
- **첫/마지막 버튼 클릭**: 첫/마지막 페이지로 이동
- **Tab**: 버튼/링크 간 포커스 이동

### 플랫폼 고려사항

- 모바일에서 터치 영역을 충분히 확보한다 (최소 44x44px).
- 모바일에서 첫/마지막 버튼을 숨기고 이전/다음 버튼만 표시할 수 있다.

## 탭바 (Tab Bar)

### 용도

모바일 환경에서 화면 하단에 고정되는 내비게이션 바. 앱의 주요 섹션 간 빠른 전환을 제공한다.

### 구조

- 탭바 컨테이너
  - 탭 항목 (3~5개)
    - 아이콘
    - 레이블 텍스트

### 사용성 가이드라인

1. 탭바는 모바일 하단 네비게이션 용도로 사용한다. PC에서는 메인 메뉴를 사용한다.
2. 주요 섹션 3~5개만 배치한다. 5개를 초과하지 않는다.
3. 각 탭에 아이콘과 텍스트 레이블을 함께 표시한다.
4. 현재 활성 탭을 시각적으로 구분한다 (색상, 아이콘 스타일 변경).
5. 모든 화면에서 탭바가 하단에 고정되어 표시된다.

### 접근성 가이드라인

- `role="navigation"` 또는 `<nav>` 태그를 사용한다.
- 현재 활성 탭에 `aria-current="page"`를 적용한다.
- 아이콘에 `aria-hidden="true"`, 텍스트 레이블이 접근 가능한 이름을 제공한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.5 Multiple Ways

### 상호작용

- **탭 클릭**: 해당 섹션으로 이동
- **탭 길게 누르기**: 없음 (단순 탭 동작만)

### 플랫폼 고려사항

- PC에서는 탭바를 사용하지 않는다. 모바일 전용 컴포넌트이다.
- 하단 고정 시 `position: fixed; bottom: 0`을 사용하고, 본문에 하단 여백을 확보한다.
