name: krds-verify
description: Use when verifying, auditing, or testing a Korean government digital service against KRDS UI/UX guidelines. Runs 311-item checklist with P/F/E/N/A grading. Triggers on 검증, 체크리스트, 품질 검사, KRDS 준수, 감사, 점검, QA, 심사.
---

## 1. Overview

311개 체크리스트 기반 KRDS UI/UX 가이드라인 자동 검증 도구.
4개 대분류: 스타일(21) + 컴포넌트(170) + 기본 패턴(47) + 서비스 패턴(73) = 311개

## 2. Iron Law

```
NO DEPLOYMENT WITHOUT VERIFICATION
구현이 완료되면 반드시 검증을 실행하세요.
```

## 3. 검증 프로세스

```
준비 → 수행 → 결과서 → 개선
```

## 4. 검증 준비

- krds-onboarding에서 설정한 기관 유형 확인
- 기관 유형별 적용 범위:
  - 정부 상징: 공통 295개 + 정부 상징 전용 8개(스타일) + 2개(컴포넌트) = 305개
  - 자체 상징: 공통 295개 + 자체 상징 전용 6개(스타일) = 301개
- 검증 대상 화면 최소 10개 선정 요청
- 검증 환경:
  - OS: Windows 10+ (필수), Mac OS Monterey+ (권장)
  - 해상도: 1920 x 1080 (FHD)
  - 브라우저: Chrome 최신
  - 확대/축소: 100%
  - 입력: 키보드(Tab, Enter, Space), 마우스
  - 확장 프로그램 미설치, 캐시/쿠키 삭제

## 5. 검증 수행

4개 카테고리를 서브에이전트로 병렬 검증 가능:

| 에이전트 | 범위 | 항목 수 |
|----------|------|---------|
| style-verifier | 가. 스타일 (색상/서체/형태) | 21개 |
| component-verifier | 나. 컴포넌트 (20개 세부) | 170개 |
| pattern-verifier | 다. 기본 패턴 (4개 세부) | 47개 |
| service-verifier | 라. 서비스 패턴 (20개 세부) | 73개 |

각 항목을 대상 화면에서 확인하고 판정.

## 6. 판정 기준

| 판정 | 의미 | 조건 |
|------|------|------|
| **P (Pass)** | 준수 | 준수 기준의 모든 절차가 True |
| **F (Fail)** | 미준수 | 하나라도 미충족, 또는 미준수 사례 해당 |
| **E (예외)** | 예외 처리 | 본래 대상이나 예외 가능 |
| **N/A** | 해당 없음 | 검증 대상 아님 |

규칙:
- 준수 기준의 모든 번호가 True → P
- 하나라도 False → F
- 미준수 사례에 해당 → 준수 기준 무관하게 F
- 하나의 가이드라인이 여러 체크리스트로 분리된 경우, 모든 체크리스트 충족해야 준수

## 7. 결과 산출

```
준수율 = P / (P + F) × 100%
(E, N/A는 분모에서 제외)
```

## 9. 개선 안내

F 항목에 대해:
- 스타일 관련 → krds-style 스킬 참조
- 컴포넌트 관련 → krds-components 스킬 참조
- 패턴 관련 → krds-patterns 스킬 참조

## 10. Integration

- **Called by**: 구현 완료 후 사용자가 직접 호출 또는 krds-patterns 이후 자동 안내
- **Followed by**: krds-license (검증 통과 후 저작권 확인)
