# 서비스 패턴 (Service Patterns)

특정 과업 흐름 전체를 다루는 5개 서비스 패턴입니다. 각 패턴은 단계별 플로(flow)로 구성되며, 가이드라인은 필수(Do)/권장(Better)/우수(Best) 수준으로 구분됩니다.

---

## 1. 방문

사용자가 서비스의 메인 화면에 도착했을 때의 패턴입니다.

### 플로
메인 화면 도착

### 가이드라인 (3개, 필수)

1. **시급/중요 정보 빠르게 인지** -- 사용자가 메인 화면에 도착했을 때 시급하거나 중요한 정보를 빠르게 인지할 수 있도록 배치합니다.
2. **최신 정보** -- 메인 화면에 표시되는 정보는 항상 최신 상태를 유지합니다.
3. **정확한 정보** -- 메인 화면에 표시되는 정보는 정확해야 합니다.

### 추천 메인 페이지 섹션 구조 (공정거래위원회 참조)

공정위(ftc.go.kr)는 KRDS 가이드라인을 적용한 대표적 성공 사례입니다. 아래 6단 구조를 추천합니다.

**섹션 1: 공식 배너**
- `html/code/masthead.html` 참조
- "이 누리집은 대한민국 공식 전자정부 누리집입니다." 텍스트
- 모든 화면 최상단 배치

**섹션 2: 헤더**
- `html/code/header.html` 참조
- 유틸리티 링크 (우상단, 4개 이내): 누리집안내 | 국가상징 | ENGLISH | 어린이 | 화면크기 | 관련 사이트
- 로고 (좌상단) + 통합검색 (우측) + 로그인
- 메인 메뉴 (6~7개 1depth): 소식·뉴스 | 정책·제도·자료 | 정보공개 | 심결·법령 | 민원·참여 | 해외경쟁정책 | 기관소개

**섹션 3: 주요소식 슬라이드쇼 (핵심 영역)**

이 섹션이 메인 페이지의 시각적 중심입니다.

```
구조: 텍스트(좌 43%) + 이미지(우 56%)
높이: 406px
전환: Swiper fade (opacity, 400ms)
자동재생: 4초 간격
인디케이터: bullets + 재생/멈춤 + 이전/다음
```

텍스트 영역:
- 제목 (`.text-title`): Heading large 32px, bold, gray-90
- 본문 (`.text-cont`): Body large 19px, regular, gray-60
- "자세히 보러가기" 링크

이미지 영역:
- `background-image: url(...)` + `background-size: cover`
- `border-radius: 12px` (KRDS xlarge)
- 높이 = 섹션 전체 높이 406px

상세 HTML/CSS/JS → krds-components 스킬의 `references/layout.md` 캐러셀 섹션 참조

**섹션 4: 자주찾는 서비스 퀵메뉴**
- 아이콘 + 텍스트 그리드 (가로 한 줄 6~9개)
- 각 항목: SVG 아이콘 48~64px + 서비스명 텍스트
- 예시: 나의 민원조회 | 방문접수예약 | 불공정거래신고 | 익명제보 | 사업자 정보공개 | 의결서 조회
- 모바일: 2열 그리드

**섹션 5: 소식 탭**
- KRDS 탭 컴포넌트 (`html/code/tab.html`) 사용
- 3~7개 탭: 최근소식 | 보도자료 | 공지사항 | 채용 | 심의안건 등
- 각 탭 패널에 최신 게시물 4~5개 (제목 + 날짜)
- "더보기" 링크

**섹션 6: 알림판 + SNS + 배너**

| 영역 | 구성 |
|------|------|
| 알림판 | 이미지 배너 캐러셀 (2~4개) |
| SNS | 탭 (유튜브/인스타/블로그) + 최신 포스트 3개 썸네일 + SNS 바로가기 아이콘 |
| 배너 | 관련 사업/서비스 배너 |

**섹션 7: 푸터**
- `html/code/footer.html` 참조
- 상단: 관련 사이트 드롭다운 (업무별 누리집, 산하기관, 관련 누리집)
- 기관 로고 + 주소 + 대표전화 + 팩스
- 정책 링크: 개인정보처리방침(강조) | 저작권 정책 | 웹 접근성 인증
- 저작권: © 기관명. All rights reserved.

### 섹션 간 간격

| 구간 | PC | 모바일 |
|------|-----|--------|
| 섹션 간 | 64px (gap-11) | 40px (gap-9) |
| 콘텐츠 래퍼 | max-width: 1200px, padding: 0 24px | padding: 0 16px |

## 2. 검색

사용자가 원하는 정보를 검색을 통해 찾는 전체 흐름입니다.

### 플로 (6단계)
기능 찾기 → 검색어 입력 → 결과 확인 → 결과 이용 → 재검색 → 종료

### 가이드라인 (23개)

#### 단계 1: 기능 찾기 (1개)

1. **통합 검색, 모든 화면에서 실행 가능** -- 통합 검색 기능을 제공하며, 어떤 화면에서든 검색을 실행할 수 있어야 합니다.

#### 단계 2: 검색어 입력 (10개)

1. **실행+삭제 버튼** -- 검색 실행 버튼과 입력 내용 삭제 버튼을 모두 제공합니다.
2. **스코프 필터 '전체'** -- 검색 범위(스코프) 필터의 기본값은 '전체'로 설정합니다.
3. **기본 옵션 '전체'** -- 검색 옵션의 기본값은 '전체'로 설정합니다.
4. **레이블** -- 검색 입력 필드에 명확한 레이블을 제공합니다.
5. **검색 대상/방법 명확** -- 무엇을 검색할 수 있는지, 어떻게 검색하는지 명확히 안내합니다.
6. **플레이스홀더 대체 금지** -- 플레이스홀더 텍스트가 레이블을 대체하지 않도록 합니다.
7. **실행 상호작용** -- Enter 키와 검색 버튼 클릭 모두로 검색을 실행할 수 있어야 합니다.
8. **실시간 도움 출현** -- 검색어 입력 시 자동완성/추천 검색어가 실시간으로 표시됩니다.
9. **실시간 도움 탐색** -- 키보드 방향키로 자동완성 목록을 탐색할 수 있어야 합니다.
10. **실시간 도움 실행** -- 자동완성 항목을 선택하여 바로 검색을 실행할 수 있어야 합니다.

#### 단계 3: 결과 확인 (9개)

1. **검색어 입력 필드 유지** -- 결과 화면에서도 검색어 입력 필드를 유지합니다.
2. **검색어 유지** -- 결과 화면에서 기존에 입력한 검색어가 유지되어야 합니다.
3. **결과 수 표시** -- 검색 결과의 총 건수를 표시합니다.
4. **탭별 결과 수** -- 검색 결과가 탭으로 분류될 경우 각 탭별 결과 수를 표시합니다.
5. **없음 상태 정보** -- 검색 결과가 없을 때 명확한 빈 상태(empty state) 메시지를 표시합니다.
6. **없음 시 대안 안내** -- 검색 결과가 없을 때 검색어 수정, 다른 키워드 등 대안을 안내합니다.
7. **기간 필터** -- 결과에 기간 필터를 제공하여 날짜 범위로 결과를 좁힐 수 있도록 합니다.
8. **정렬 컨트롤** -- 검색 결과의 정렬(관련도순, 최신순 등) 컨트롤을 제공합니다.
9. **스코프 범위 표시** -- 현재 검색 범위(스코프)가 무엇인지 결과 화면에 표시합니다.

#### 단계 4: 결과 이용 (2개)

1. **제목, 설명, 인덱스** -- 각 검색 결과 항목에 제목, 설명, 인덱스(번호)를 제공합니다.
2. **일치 항목 강조** -- 검색어와 일치하는 부분을 시각적으로 강조(하이라이트)합니다.

#### 단계 5: 재검색 (2개)

1. **상세 검색 숨기지 않기** -- 상세 검색(고급 검색) 기능을 숨기지 않고 접근할 수 있도록 합니다.
2. **조건 유지** -- 재검색 시 이전에 설정한 검색 조건을 유지합니다.

#### 단계 6: 종료 (1개)

1. **다른 탐색 수단 접근** -- 검색 외에 다른 탐색 수단(메뉴, 카테고리 등)으로 쉽게 접근할 수 있어야 합니다.

## 3. 로그인

사용자가 서비스에 인증하는 전체 흐름입니다.

### 플로 (4단계)
찾기 → 방식 확인 → 정보 입력 → 이용

### 가이드라인 (14개)

#### 단계 1: 찾기 (3개)

1. **일관된 위치** -- 로그인 진입점을 모든 페이지에서 일관된 위치에 배치합니다.
2. **항상 로그인 화면 연결** -- 로그인 진입점은 항상 로그인 화면으로 직접 연결되어야 합니다.
3. **모바일 숨기지 않기** -- 모바일 환경에서도 로그인 진입점을 숨기지 않습니다.

#### 단계 2: 방식 확인 (2개)

1. **도움말+회원가입 링크** -- 로그인 방식 선택 화면에서 도움말과 회원가입 링크를 제공합니다.
2. **다양한 환경 도움말** -- 다양한 기기·브라우저 환경에서의 로그인 방법에 대한 도움말을 제공합니다.

#### 단계 3: 정보 입력 (9개)

1. **도움말+회원가입** -- 로그인 정보 입력 화면에서도 도움말과 회원가입 링크를 제공합니다.
2. **아이디 형식 단서** -- 아이디 입력 필드에 형식 단서(예: 이메일, 숫자 등)를 제공합니다.
3. **비밀번호 기본 숨김** -- 비밀번호 입력값은 기본적으로 마스킹 처리합니다.
4. **비밀번호 표시 옵션** -- 비밀번호를 일시적으로 표시할 수 있는 토글 옵션을 제공합니다.
5. **복붙 허용** -- 비밀번호 필드에 복사·붙여넣기를 허용합니다.
6. **아이디/비밀번호 찾기** -- 아이디 찾기, 비밀번호 찾기 링크를 제공합니다.
7. **오류 메시지 명확** -- 로그인 실패 시 구체적이고 명확한 오류 메시지를 표시합니다.
8. **제출 전 유효성 검사** -- 로그인 버튼 클릭 전에 입력값의 형식을 유효성 검사합니다.
9. **오류 시 해당 필드 포커스** -- 오류 발생 시 해당 입력 필드로 포커스를 이동합니다.

#### 단계 4: 이용 (3개)

1. **세션 만료 전 안내** -- 세션이 만료되기 전에 사용자에게 안내합니다.
2. **만료 20초 전 모달** -- 세션 만료 20초 전에 모달 대화상자로 알림합니다.
3. **연장+로그아웃 버튼** -- 세션 만료 안내 모달에 세션 연장과 로그아웃 버튼을 함께 제공합니다.

## 4. 신청

사용자가 서비스를 신청·접수하는 전체 흐름입니다.

### 플로 (7단계)
탐색 → 정보 확인 → 유의/자격 → 작성 → 확정 → 완료 → 결과

### 가이드라인 (25개)

#### 단계 1: 탐색 (3개)

1. **신청 상태 명확** -- 신청 가능/마감/대기 등 현재 신청 상태를 명확하게 표시합니다.
2. **공식 명칭** -- 신청 항목의 공식 명칭을 사용합니다.
3. **말줄임표 미사용** -- 신청 항목 이름을 말줄임표(...)로 자르지 않습니다.

#### 단계 2: 정보 확인 (8개)

1. **어려운 단어 설명** -- 전문 용어나 어려운 단어에 대한 설명을 제공합니다.
2. **정확한 정보** -- 안내 정보는 정확해야 합니다.
3. **최신 정보** -- 안내 정보는 최신 상태를 유지해야 합니다.
4. **모든 채널 안내** -- 온라인 외에 오프라인 등 모든 신청 채널을 안내합니다.
5. **제약사항 안내** -- 신청 자격, 기간, 제한 등 제약사항을 안내합니다.
6. **필요 서식/서류** -- 신청에 필요한 서식과 구비 서류를 안내합니다.
7. **처리 절차** -- 신청 후 처리 절차와 소요 시간을 안내합니다.
8. **기본 정보 첫 번째 탭** -- 기본 정보를 첫 번째 탭에 배치합니다.

#### 단계 3: 유의/자격 (1개)

1. **작성 중단 상황 안내** -- 작성 도중 중단해야 하는 상황(임시저장, 세션 만료 등)에 대한 안내를 제공합니다.

#### 단계 4: 작성 (6개)

1. **단계 표시기** -- 신청서가 여러 단계로 구성된 경우 현재 위치를 보여주는 단계 표시기(step indicator)를 제공합니다.
2. **탭/아코디언 구분 금지** -- 신청서 작성 화면에서 탭이나 아코디언으로 입력 영역을 숨기지 않습니다.
3. **값 변경 영향 안내** -- 특정 값을 변경할 때 다른 항목에 미치는 영향을 안내합니다.
4. **문의처 제공** -- 작성 중 문의할 수 있는 연락처를 제공합니다.
5. **도움말 정확** -- 작성 도움말의 내용이 정확해야 합니다.
6. **초기화 버튼 구분** -- 초기화(리셋) 버튼을 제출 버튼과 시각적으로 명확히 구분합니다.

#### 단계 5: 확정 (3개)

1. **확인/확정 단계 필수** -- 최종 제출 전에 확인·확정 단계를 반드시 제공합니다.
2. **취소/수정 불가 안내** -- 제출 후 취소나 수정이 불가능한 사항을 명확히 안내합니다.
3. **취소 수단** -- 확정 단계에서 취소하고 이전으로 돌아갈 수 있는 수단을 제공합니다.

#### 단계 6: 완료 (1개)

1. **완료 단계 필수** -- 신청이 성공적으로 접수되었음을 알리는 완료 단계를 반드시 제공합니다.

#### 단계 7: 결과 (3개)

1. **상세 내역** -- 신청 결과의 상세 내역을 확인할 수 있어야 합니다.
2. **이력** -- 신청 이력(접수, 심사, 승인 등)을 확인할 수 있어야 합니다.
3. **현재 진행 상태** -- 현재 진행 상태를 실시간으로 확인할 수 있어야 합니다.

## 5. 정책 정보 확인

사용자가 정부 정책 정보를 탐색하고 확인하는 흐름입니다.

### 플로 (3단계)
탐색 → 정보 확인 → 자료 탐색

### 가이드라인

#### 필수 (2개)

1. **정확한 정보** -- 정책 정보는 정확해야 합니다.
2. **최신 정보** -- 정책 정보는 최신 상태를 유지해야 합니다.

#### 권장

- **필터링/검색** -- 정책 목록에서 필터링 및 검색 기능을 제공합니다.
- **분류 체계** -- 정책을 주제, 대상, 분야 등 체계적으로 분류합니다.
- **신규 배지** -- 신규 정책에 "새 글" 배지를 표시합니다.
- **문의처** -- 정책 담당 부서의 문의처를 제공합니다.
- **관련 링크** -- 관련 정책, 법령, 사업 등의 링크를 함께 제공합니다.
