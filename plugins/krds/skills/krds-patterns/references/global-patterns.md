# 기본 패턴 (Global Patterns)

페이지 유형에 관계없이 공통으로 적용되는 11개 패턴입니다.

---

## 1. 목록 탐색

### 용도
정보 항목을 목록 형태로 제공하여 사용자가 원하는 항목을 탐색할 수 있도록 합니다.

### 구조
- 구조화 목록: 제목 + 메타데이터 + 배지로 구성된 항목을 나열
- 페이지네이션: 목록 하단에 페이지 이동 컨트롤 제공

### 관련 컴포넌트
- 구조화 목록 (Structured List)
- 배지 (Badge)
- 페이지네이션 (Pagination)

---

## 2. 필터링/정렬

목록 데이터를 사용자가 원하는 조건으로 좁히거나 순서를 변경할 수 있도록 합니다.

### 가이드라인 (11개)

1. **필터링/정렬 컨트롤 숨기지 않기** -- 필터링·정렬 컨트롤은 항상 노출하여 사용자가 즉시 접근할 수 있어야 합니다.
2. **선택된 옵션 명확 구분** -- 현재 적용 중인 필터/정렬 옵션을 시각적으로 명확하게 구분합니다.
3. **적용 시각적 단서** -- 필터가 적용되었을 때 결과 영역에 시각적 변화(결과 수 갱신 등)를 제공합니다.
4. **실행 버튼 레이블** -- 필터 실행 버튼의 레이블은 동작을 명확히 나타내야 합니다 (예: "검색", "적용").
5. **그룹 헤딩** -- 필터 옵션이 여러 그룹인 경우 각 그룹에 명확한 헤딩을 부여합니다.
6. **간결 정확 레이블** -- 각 필터 옵션의 레이블은 간결하고 정확해야 합니다.
7. **관련 필터 군집화** -- 서로 관련 있는 필터 옵션은 근접하게 배치하여 군집화합니다.
8. **정렬은 우상단** -- 정렬 컨트롤은 목록 우상단에 배치합니다.
9. **날짜/시간 범위 입력** -- 날짜·시간 필터는 범위 입력(시작일~종료일)을 지원합니다.
10. **한 번에 해제 기능** -- 적용된 모든 필터를 한 번에 초기화할 수 있는 기능을 제공합니다.
11. **축약 시 필터링 우선** -- 화면이 좁아져 컨트롤을 축약해야 할 때, 정렬보다 필터링을 우선 노출합니다.

---

## 3. 상세 정보 확인

목록에서 선택한 항목의 세부 내용을 확인하는 패턴입니다.

### 가이드라인

- **콘텐츠 내 탐색 연동** -- 긴 상세 페이지에서 목차나 앵커 링크를 통해 콘텐츠 내부를 탐색할 수 있도록 합니다.
- **부가 정보 접기/펼치기** -- 보조적인 정보는 아코디언 등으로 접어두고, 사용자가 필요할 때 펼칠 수 있도록 합니다.

---

## 4. 첨부파일

파일을 다운로드하거나 첨부와 관련된 정보를 제공하는 패턴입니다.

### 가이드라인 (3개)

1. **다운로드 사전 안내** -- 파일 다운로드 전에 사용자에게 무엇이 다운로드되는지 안내합니다.
2. **크기+형식 표시** -- 첨부파일의 크기와 파일 형식(확장자)을 표시합니다.
3. **인라인 링크는 주변 텍스트 크기** -- 본문 텍스트 내 인라인으로 배치된 첨부파일 링크는 주변 텍스트와 동일한 크기로 표시합니다.

## 5. 개인 식별 정보 입력

이름, 생년월일, 성별, 전화번호 등 개인 식별 정보를 입력받는 패턴입니다.

### 가이드라인 (22개)

#### 공통
1. **용도 명확 설명** -- 개인 식별 정보를 수집하는 이유와 용도를 명확하게 설명합니다.
2. **복붙 허용** -- 사용자가 입력 필드에 값을 복사·붙여넣기 할 수 있어야 합니다.
3. **기본값 미설정** -- 개인 식별 정보 필드에 기본값을 미리 설정하지 않습니다.
4. **플레이스홀더 구분** -- 플레이스홀더 텍스트가 실제 입력값과 시각적으로 명확히 구분되어야 합니다.

#### 이름
5. **이름: 모든 문자 지원** -- 한글, 영문, 한자 등 모든 문자를 입력할 수 있도록 지원합니다.
6. **이름: 입력 형식 설명** -- 이름 입력 형식(예: 성+이름, 이름만 등)을 명확히 설명합니다.
7. **이름: 글자 수 적절** -- 최대 글자 수 제한을 충분히 넉넉하게 설정합니다.
8. **이름: 너비 = 최대 글자 수** -- 입력 필드의 너비를 최대 글자 수에 맞춥니다.
9. **이름: 단일 필드** -- 성과 이름을 분리하지 않고 단일 필드로 제공합니다.
10. **이름: 개명 도움말** -- 개명한 사용자를 위한 도움말 또는 안내를 제공합니다.

#### 생년월일
11. **생년월일: 날짜선택기 기본 금지** -- 생년월일 입력에 달력 형태의 날짜선택기를 기본으로 사용하지 않습니다.
12. **생년월일: 연도/일 셀렉트 금지** -- 연도와 일자를 드롭다운 셀렉트로 제공하지 않습니다.
13. **생년월일: 형식 안내** -- 입력 형식(예: YYYY-MM-DD)을 명확히 안내합니다.

#### 성별
14. **성별: 기본값 금지** -- 성별 필드에 기본값을 설정하지 않습니다.
15. **성별: 선택 안 함 옵션** -- "선택 안 함" 또는 이에 준하는 옵션을 제공합니다.

#### 전화번호
16. **전화번호: 특정 유형 필수 금지** -- 특정 유형(예: 휴대전화만)의 전화번호를 필수로 요구하지 않습니다.
17. **전화번호: 명확 레이블** -- 전화번호 필드의 레이블을 명확하게 표시합니다.
18. **전화번호: 다양한 문자 지원** -- 하이픈, 괄호, 공백 등 다양한 입력 형식을 허용합니다.
19. **전화번호: 형식 안내** -- 전화번호 입력 형식을 안내합니다.
20. **전화번호: 국가코드 콤보박스** -- 국제전화 국가코드를 콤보박스로 제공합니다.
21. **전화번호: 복수 시 구분 레이블** -- 여러 전화번호를 입력받을 때 각각의 용도를 구분하는 레이블을 제공합니다.
22. **전화번호: 주 번호 지정 옵션** -- 복수의 전화번호 중 주 연락처를 지정할 수 있는 옵션을 제공합니다.

## 6. 입력폼

사용자로부터 데이터를 입력받는 폼 전반의 구성 패턴입니다.

### 가이드라인 (11개)

1. **제목 제공** -- 입력폼의 제목(무엇을 입력하는 폼인지)을 명확히 제공합니다.
2. **단일 열 수직 정렬** -- 입력 필드를 단일 열(single-column)로 수직 정렬합니다.
3. **충분한 간격** -- 입력 필드 사이에 충분한 간격을 확보하여 시각적으로 분리합니다.
4. **액션 버튼 일관된 위치** -- 제출, 취소 등 액션 버튼은 폼 내에서 일관된 위치에 배치합니다.
5. **비활성 제출 버튼 사용** -- 필수 항목이 모두 입력되기 전까지 제출 버튼을 비활성 상태로 유지합니다.
6. **레이블+설명** -- 각 입력 필드에 레이블과 필요 시 보조 설명을 함께 제공합니다.
7. **입력값 단위 표시** -- 숫자 입력 시 단위(원, kg, cm 등)를 명확히 표시합니다.
8. **필수/선택 구분** -- 필수 입력 항목과 선택 입력 항목을 명확히 구분합니다.
9. **일관된 구분 방식** -- 필수/선택 구분 표시 방식(별표, 텍스트 등)을 폼 전체에서 일관되게 사용합니다.
10. **자동완성** -- 브라우저 자동완성(autocomplete) 속성을 적절히 활용합니다.
11. **모바일 고려** -- 모바일 환경에서 적절한 키보드 타입(숫자, 이메일 등)이 표시되도록 합니다.

## 7. 동의

약관 동의, 개인정보 수집·이용 동의 등 사용자 동의를 받는 패턴입니다.

### 가이드라인

- **전체 동의/개별 동의 구분** -- 전체 동의와 개별 항목 동의를 구분하여 제공합니다.
- **필수/선택 동의 명확** -- 필수 동의 항목과 선택 동의 항목을 명확하게 구분합니다.

### 관련 컴포넌트
- 체크박스 (Checkbox)

---

## 8. 도움

사용자가 서비스 이용 중 도움이 필요할 때 활용하는 패턴입니다.

### 가이드라인

- **문의처 제공** -- 전화, 이메일, 챗봇 등 문의할 수 있는 수단을 제공합니다.
- **맥락적 도움말** -- 사용자가 현재 수행 중인 과업에 맞는 도움말을 해당 화면에서 바로 확인할 수 있도록 합니다.
- **FAQ** -- 자주 묻는 질문(FAQ)을 제공하여 사용자가 스스로 문제를 해결할 수 있도록 합니다.

### 관련 컴포넌트
- 도움 패널 (Help Panel)
- 맥락적 도움말 (Contextual Help)
- 코치마크 (Coach Mark)

---

## 9. 사용자 피드백

시스템이 사용자에게 상태 정보를 전달하는 패턴입니다.

### 가이드라인

- **성공/오류/경고/정보 시스템 색상 적용** -- 시스템 피드백 유형(성공, 오류, 경고, 정보)에 따라 KRDS 시스템 색상을 적용합니다.
  - 성공: Success 색상
  - 오류: Danger 색상
  - 경고: Warning 색상
  - 정보: Information 색상
- **아이콘+텍스트 병행** -- 색상만으로 구분하지 않고, 아이콘과 텍스트를 함께 제공하여 접근성을 확보합니다.

---

## 10. 오류

사용자 입력 오류 또는 시스템 오류를 처리하는 패턴입니다.

### 가이드라인

- **구체적 오류 메시지** -- "입력 오류" 같은 모호한 메시지 대신, 무엇이 잘못되었고 어떻게 수정할 수 있는지 구체적으로 안내합니다.
- **해당 필드로 포커스 이동** -- 오류 발생 시 해당 입력 필드로 포커스를 자동 이동합니다.
- **인라인 검증** -- 사용자가 입력을 완료한 시점에 즉시 인라인으로 유효성을 검증하여 피드백합니다.
- **시스템 색상 Danger 사용** -- 오류 상태에는 KRDS 시스템 색상 중 Danger 색상을 사용합니다.

---

## 11. 확인

사용자가 최종 제출 전에 입력 내용을 확인하고 검토하는 패턴입니다.

### 가이드라인

- **제출 전 핵심 정보 요약** -- 제출 직전에 사용자가 입력한 핵심 정보를 요약하여 보여줍니다.
- **취소 수단 제공** -- 제출을 취소하고 이전 단계로 돌아갈 수 있는 수단을 제공합니다.
- **수정 불가 사항 명확 안내** -- 제출 후 수정이 불가능한 항목이 있다면 이를 명확히 안내합니다.
