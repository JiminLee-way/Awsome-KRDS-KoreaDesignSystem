# Identity 컴포넌트

## 공식 배너 (Masthead)

### 용도

정부 공식 웹사이트임을 나타내는 최상단 배너. 사용자에게 해당 사이트가 대한민국 정부 기관의 공식 서비스임을 알린다.

### HTML 스니펫

```
html/code/masthead.html
```

### 구조

- 배너 컨테이너 (`krds-masthead`)
- 정부 로고/마크
- 공식 사이트 안내 텍스트

### 사용성 가이드라인

1. 모든 화면의 최상단에 배치한다. 헤더보다 위에 위치해야 한다.
2. 배너가 지나치게 시각적 주의를 끌지 않도록 절제된 스타일을 유지한다.
3. 배너의 텍스트, 색상, 스타일을 임의로 변형하지 않는다. KRDS에서 제공하는 원본 그대로 사용한다.
4. 비정부 사이트에서는 공식 배너를 사용하지 않는다.

### 접근성 가이드라인

- 건너뛰기 링크(`skip_link`)는 공식 배너보다 이전에 위치시킨다. 키보드 사용자가 배너를 건너뛸 수 있어야 한다.
- 배너 내 이미지에는 적절한 `alt` 텍스트를 제공한다.
- KWCAG 2.2 / WCAG 2.1 준수: 1.1.1 Non-text Content, 2.4.1 Bypass Blocks

### 상호작용

- 정적 컴포넌트로 별도의 상호작용은 없다.

### 플랫폼 고려사항

- 모바일에서도 동일한 배너를 최상단에 표시한다. 텍스트 크기만 반응형으로 조정된다.

## 운영기관 식별자 (Identifier)

### 용도

사이트를 운영하는 정부 기관의 상위 기관 로고와 정보를 표시한다. 주로 푸터 영역에 배치된다.

### HTML 스니펫

```
html/code/identifier.html
```

### 구조

- 식별자 컨테이너 (`krds-identifier`)
- 상위 기관 로고(들)
- 안내 텍스트

### 사용성 가이드라인

1. 운영 기관의 상위 기관 로고를 표시한다. 직접 운영 기관이 아닌 상위 관할 기관의 로고를 사용한다.
2. 복수의 상위 기관이 있을 경우, 계층 순서대로(상위→하위) 로고를 배치한다.
3. 안내 텍스트는 모든 화면에서 일관된 위치에 배치한다.

### 접근성 가이드라인

- 식별자는 푸터(`<footer>`) 내부에 배치한다.
- 모든 기관 로고 이미지에 `alt` 텍스트를 제공한다 (예: `alt="국무조정실"`).
- KWCAG 2.2 / WCAG 2.1 준수: 1.1.1 Non-text Content

### 상호작용

- 로고 클릭 시 해당 기관의 공식 사이트로 이동한다 (새 탭).

### 플랫폼 고려사항

- 모바일에서 로고가 여러 개일 경우 줄바꿈하여 표시한다.

## 헤더 (Header)

### 용도

사이트의 최상단 내비게이션 영역. 로고, 메인 메뉴, 유틸리티 링크, 검색 등 주요 탐색 요소를 포함한다.

### HTML 스니펫

```
html/code/header.html
```

### 구조

- 헤더 컨테이너 (`krds-header`)
  - 로고 영역
  - 메인 내비게이션 (메인 메뉴)
  - 유틸리티 영역 (로그인, 회원가입, 언어 전환 등)
  - 검색 영역
  - 모바일 햄버거 버튼

### 사용성 가이드라인

1. 헤더의 기본 스타일 수정을 최소화한다. KRDS 기본 구조를 유지한다.
2. 로고는 좌상단에 배치한다. 로고 클릭 시 메인 페이지로 이동한다.
3. 헤더 내부 컴포넌트(로고, 메뉴, 유틸리티, 검색)는 일관된 순서로 배치한다.
4. 유틸리티 링크(로그인, 회원가입 등)는 우상단에 배치한다.
5. 유틸리티 링크 사이는 디바이더(구분선)로 시각적으로 구분한다.
6. 유틸리티 항목이 5개 이상인 경우 드롭다운으로 묶어 제공한다.
7. 언어 전환 드롭다운에 국기 아이콘을 사용하지 않는다. 언어 이름(텍스트)만 표시한다.
8. 아이콘에는 반드시 텍스트 레이블을 함께 표시한다. 아이콘만 단독으로 사용하지 않는다.
9. 스크롤 시 헤더 고정 규칙: 헤더가 2행을 초과하는 경우, 스크롤 다운 시 숨기고 스크롤 업 시에만 다시 표시한다.

### 접근성 가이드라인

- 건너뛰기 링크(`skip_link`)를 헤더 최상단(첫 번째 포커스 요소)에 배치한다.
- 로고 이미지에 `alt` 텍스트를 제공한다 (예: `alt="기관명 로고"`).
- 내비게이션 영역에 `<nav>` 태그와 `aria-label`을 사용한다.
- 검색 입력에 `aria-label="사이트 검색"`을 제공한다.
- KWCAG 2.2 / WCAG 2.1 준수: 2.4.1 Bypass Blocks, 2.4.2 Page Titled, 3.2.3 Consistent Navigation

### 상호작용

- **로고 클릭**: 메인 페이지로 이동
- **유틸리티 링크 클릭**: 해당 기능 페이지로 이동
- **햄버거 버튼 클릭**: 모바일 메뉴 열기/닫기
- **검색 아이콘 클릭/Enter**: 검색 실행

### 플랫폼 고려사항

- 모바일에서는 메인 메뉴를 햄버거 메뉴로 전환한다.
- 모바일 헤더는 로고와 햄버거 버튼만 표시하고, 유틸리티 링크는 모바일 메뉴 내부로 이동한다.
- 참조: `main_menu_mobile.html`

## 푸터 (Footer)

### 용도

사이트의 최하단 영역. 기관 정보, 연락처, 정책 링크, 저작권 등을 제공한다.

### HTML 스니펫

```
html/code/footer.html
```

### 구조

- 푸터 컨테이너 (`krds-footer`)
  - 기관 로고
  - 연락처 정보 (주소, 전화, 이메일)
  - 유틸리티 링크 (관련 사이트 등)
  - 정책 링크 (개인정보처리방침, 이용약관 등)
  - 저작권 표시

### 사용성 가이드라인

1. 기관 로고, 연락처 정보, 저작권 표시를 반드시 제공한다.
2. 모든 화면에서 일관된 위치(최하단)에 푸터를 배치한다.
3. 푸터 내 정보는 일관된 순서로 배치한다: 로고 → 연락처 → 유틸리티 링크 → 정책 링크 → 저작권.
4. 개인정보처리방침 링크는 다른 정책 링크와 시각적으로 구분하여 반드시 강조 표시한다 (굵은 글씨 등).
5. 전자상거래 사이트의 경우 이용약관 링크를 필수로 포함한다.
6. 사용자가 빈번하게 접근하는 링크를 푸터에 배치한다.
7. 푸터의 내비게이션 구조는 기본 내비게이션(헤더 메뉴)과 동일한 구조를 유지한다.
8. 연락처 정보는 항상 최신 상태를 유지한다.
9. 모든 링크가 정확한 목적지로 연결되는지 확인한다.
10. 화면 크기에 관계없이 동일한 정보 순서를 유지한다.
11. 푸터의 기본 스타일 수정을 최소화한다. KRDS 기본 구조를 유지한다.

### 접근성 가이드라인

- 푸터 영역은 `<footer>` 시맨틱 태그를 사용한다.
- 기관 로고 이미지에 `alt` 텍스트를 제공한다.
- 연락처 링크(전화번호, 이메일)에는 적절한 `href` 형식을 사용한다 (`tel:`, `mailto:`).
- 정책 링크 영역에 `<nav aria-label="정책 링크">`를 적용한다.
- KWCAG 2.2 / WCAG 2.1 준수: 1.1.1 Non-text Content, 2.4.4 Link Purpose, 3.2.3 Consistent Navigation

### 상호작용

- **로고 클릭**: 메인 페이지로 이동
- **연락처 링크 클릭**: 전화 발신 또는 이메일 클라이언트 열기
- **정책 링크 클릭**: 해당 정책 페이지로 이동
- **유틸리티 링크 클릭**: 관련 사이트로 이동 (새 탭)

### 플랫폼 고려사항

- 모바일에서 연락처 정보는 세로 배치로 전환한다.
- 정책 링크는 모바일에서도 모두 표시한다 (숨기지 않는다).
- 유틸리티 링크가 많을 경우 아코디언 형태로 접을 수 있다.
